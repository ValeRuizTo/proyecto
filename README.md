# Proyecto Almacén Súper Amortiguadores LTDA

Almacén Súper Amortiguadores LTDA es un negocio del sector automotriz que vende repuestos y realiza diagnósticos y reparaciones de vehículos. Actualmente, la gestión de la información se realiza de forma manual mediante un único documento en Google Docs.

Esta forma de trabajo genera problemas como falta de control de inventario, desorganización en los procesos y ausencia de trazabilidad, lo que evidencia la necesidad de implementar una solución tecnológica más estructurada y eficiente.

## Trabajo de primer corte: 

### Carpeta entrega1-BPMN

En este taller se desarrolló el modelamiento del proceso de venta e inventario de la empresa mediante notación BPMN, con el objetivo de representar de forma clara cómo funciona actualmente el flujo operativo del negocio. A partir del análisis del proceso real, se identificaron los actores, decisiones clave y puntos críticos, como la verificación de disponibilidad de repuestos y la dependencia de control manual del inventario. El resultado es un diagrama que permite visualizar tanto el funcionamiento actual como sus principales limitaciones, sirviendo como base para futuras mejoras en la gestión del negocio.


### Carpeta entrega2-Entidad Relacion (er)
En este taller se desarrolló el modelo de información del negocio mediante un diagrama Entidad-Relación (ERD) y un diagrama de contexto, con el fin de estructurar los datos y entender cómo fluye la información entre los actores del sistema. A partir del análisis del proceso previamente modelado, se definieron las entidades clave, sus relaciones y la forma en que el inventario, las compras y la facturación se integran en un sistema organizado.

A diferencia del taller anterior, en este caso se incluyeron dos perspectivas: un diagrama de contexto del estado actual, que refleja cómo opera el negocio hoy, y un modelo entidad-relación de la solución propuesta, que plantea una estructura de datos organizada para mejorar la gestión del inventario y los procesos asociados. El resultado permite evidenciar la transición desde un manejo empírico hacia un modelo estructurado, trazable y alineado con las necesidades del negocio.

## Trabajo de segundo corte: 

### Carpeta entrega4-infra

En este taller se diseñó el diagrama de infraestructura del sistema, comparando el estado actual (AS-IS) con la solución propuesta (TO-BE). Por un lado, se modeló la arquitectura actual basada en Google Docs, evidenciando su simplicidad pero también sus limitaciones, como la dependencia de un único documento y la falta de control estructurado.

Por otro lado, se planteó una arquitectura en la nube más robusta y segura, que incluye una aplicación web, servicios especializados, base de datos central y mecanismos de seguridad. Esta comparación permite visualizar claramente la evolución del sistema, pasando de un manejo manual y vulnerable a una solución organizada, escalable y con mayor control sobre la información.

### Carpeta entrega5-Seguridad

En este taller se aplicó la metodología STRIDE para analizar los riesgos de seguridad del sistema actual (AS-IS) de la empresa, el cual está basado principalmente en el uso de Google Docs. A partir de este análisis, se identificaron amenazas reales relacionadas con el acceso no autorizado, la manipulación de información, la falta de trazabilidad y la dependencia de un único documento para gestionar datos críticos del negocio.

El ejercicio permitió evidenciar las principales vulnerabilidades del sistema actual y entender cómo estas afectan directamente la operación del negocio, sirviendo como base para justificar la necesidad de una solución más segura y estructurada.

### Carpeta entrega6-Normativas

En este taller se evaluó el cumplimiento normativo del sistema de información del negocio, contrastando su estado actual con los requisitos legales y de seguridad establecidos por estándares como ISO 27001, y la legislación colombiana de protección de datos. A partir de un análisis estructurado tipo checklist, se identificaron brechas importantes en el manejo de la información, especialmente considerando que el sistema actual se basa en herramientas como Google Docs sin controles formales.
